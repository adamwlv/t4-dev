T4 Dev Setup with Grunt.js

This is a sample project I've set up for helping VCU web developers get started with effectively developing for TerminalFour, VCU's official content management system. I say effectively because the gruntfile.js that I've set up will do a number of things to make building T4 page templates easier and efficient.

  • Validates all HTML according to the W3C engine
  • Minifies all CSS
  • Strips out any unused CSS*
  • Concatenates and minifies all Javascript
  • Replaces all local image paths in any CSS with T4 media library tags
  • Automatically reloads the browser on any changes in HTML, SASS, or JS
  • Preprocesses all CSS with SASS + Compass
  • Autoprefixes any CSS3 properities for fallback support with Compass
  • Loads in Modernizr.js (a stripped version) for HTML5 support
  • Combines all media queries and adds them to the bottom of CSS

Before you get started digging into the gruntfile.js make sure you are familiar with using Grunt.js and SASS. If you would like contribute to the project feel free to make a pull request through Github.

*Build out a components guide where all components are used so the UNCSS in the gruntfile.js does not strip it out
